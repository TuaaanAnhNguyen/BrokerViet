# BrokerViet - Tech Service Marketplace Mobile App

BrokerViet is a multi-tenant mobile marketplace concept built as a university course learning project to explore mobile application development with Flutter and cloud-native backends. 

The application connects **Customers** looking for technical support with **Service Providers** offering hardware repairs, software setups, custom PC consultation, and tech equipment rentals (such as gaming rigs, VR headsets, and monitors).

---

## Project Architecture Overview

The repository is organized into distinct environments focusing on frontend presentation and backend endpoints:

*   **`frontend_flutter/`**: The core mobile application workspace built using **Flutter & Dart**. It implements the **BLoC pattern** to handle UI interactions and state separation across features like authentication, booking tracking, real-time chat, and map locations.
*   **Supabase (Cloud Backend)**: The primary database and backend infrastructure managed directly via the Supabase Web Dashboard. Core database operations, transactional checks, and background scripts are driven directly in the cloud through PostgreSQL Stored Procedures (RPCs) and Edge Functions.
*   **`backend_dotnet/`**: A minimal C# project originally intended for use during the early design phase. The team ultimately decided to pivot away from a traditional backend setup and move fully to Supabase for the database and business logic layer.
*   **`supabase/`**: Local configuration files generated by specific feature setups.

## Key Technical Features Implemented

*   **Dual-Role Flows:** Split onboarding paths and specialized operational viewports for both Customers and Service Providers.
*   **Real-Time Subscriptions:** Live in-app messaging and continuous order tracking updates managed directly via **Supabase Real-time listener streams**.
*   **Background Notifications:** Integrated database state triggers connected to **Firebase Cloud Messaging (FCM)** for push alerts.
*   **Interactive Geolocation:** GPS-based map integrations displaying nearby technicians and storefront pins with directional paths.
*   **Data Integrity:** Structured PostgreSQL foreign key rules prioritizing cascading deletions for volatile data tracks (chat streams) and `ON DELETE SET NULL` configurations for transactional logs.

## Important Disclaimer

> **Note on Backend Availability:** This project is hosted entirely on the **Supabase Free Tier**. Due to standard provider policies on free-tier projects, the backend database and Edge Functions automatically enter a paused/hibernating state after periods of inactivity. If you are testing the live application or connecting to the instance and notice loading timeouts, the database project may need a manual restoration/wake-up trigger.
